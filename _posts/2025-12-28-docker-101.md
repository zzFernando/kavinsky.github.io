## O que é Docker?

Docker é uma plataforma que permite empacotar sua aplicação e todas as suas dependências em um container. Pense nele como uma máquina virtual leve, mas muito mais rápida e eficiente.

## Conceitos fundamentais

### Image (Imagem)
Uma imagem é um template de somente leitura que contém tudo que sua aplicação precisa.

### Container
Um container é uma instância em execução de uma imagem.

### Dockerfile
Um arquivo de texto que contém instruções para construir uma imagem.

## Seu primeiro Dockerfile

```dockerfile
FROM node:18-alpine

WORKDIR /app

COPY package*.json ./

RUN npm install

COPY . .

EXPOSE 3000

CMD ["npm", "start"]
```

## Construindo a imagem

```bash
docker build -t minha-app:1.0 .
```

## Executando um container

```bash
docker run -p 3000:3000 minha-app:1.0
```

Sua aplicação agora está rodando em `http://localhost:3000`!

## Docker Compose para múltiplos containers

`docker-compose.yml`:

```yaml
version: '3.8'

services:
  app:
    build: .
    ports:
      - "3000:3000"
    environment:
      - NODE_ENV=production
    depends_on:
      - db

  db:
    image: postgres:14
    environment:
      POSTGRES_PASSWORD: senha123
    volumes:
      - db_data:/var/lib/postgresql/data

volumes:
  db_data:
```

Execute com:

```bash
docker-compose up
```

## Boas práticas

1. **Use imagens oficiais**: `node:18`, `postgres:14`, etc.
2. **Use Alpine**: Versões `.alpine` são mais leves
3. **Multi-stage builds**: Para reduzir tamanho da imagem
4. **Não use root**: Crie um usuário não-root
5. **Use `.dockerignore`**: Excluda arquivos desnecessários

## Conclusão

Docker revolucionou a forma como desenvolvemos e deployamos aplicações. Com os conceitos básicos aqui, você já consegue containerizar suas aplicações!
